# Recipe: followup -- Follow-Up Tracking Draft

## Executive Summary

Draft follow-up notes from real tracker history and verified role/CV evidence.
Agents use this to identify follow-up candidates; humans use the summary to
confirm cadence is not being invented.

Current status: draft workflow. Use this after `data/ats/applications.md` has
real application history.

## Required Reads

- `recipes/_shared.md`
- `data/ats/applications.md`
- `recipes/tracker.md`
- `logs/RUN_LOG.md`

## Phase Gates

1. **Tracker gate:** Do not draft follow-ups without tracker rows that include
   dates and statuses.
2. **Local evidence gate:** Use tracker, role report, posting, and CV evidence
   before external lookup.
3. **Stored script gate:** No maintained follow-up cadence script exists in
   `scripts/`; do not report automated cadence statistics.
4. **Verification gate:** Each follow-up must cite the status/date/reason.
5. **Logging gate:** Record sent follow-ups in tracker notes or run log.

## Workflow

1. Read active applications.
2. Identify applications with dates and statuses that support a follow-up.
3. Draft short follow-up notes from verified CV/report evidence only.
4. Record any sent follow-up as a tracker note or log entry.

## Not Implemented Yet

There is no maintained follow-up cadence script in `scripts/` yet. Until one
exists, do not report automated cadence statistics as verified.
